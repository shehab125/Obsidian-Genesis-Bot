# 📜 التقرير الشامل لتسليم مشروع: Obsidian Genesis Bot

السلام عليكم ورحمة الله وبركاته،

تم بحمد الله الانتهاء من تطوير وتفعيل **نظام Obsidian Genesis Bot المتكامل** الخاص بتعدين وعمليات عملة $OBSD$ الرقمية عبر تطبيق تليجرام المصغر (Telegram Mini App) ولوحة التحكم الإدارية (Admin Dashboard).

هذا التقرير يمثل **المستند الكامل للمشروع** ويوضح كافة الأنظمة والواجهات والميزات التقنية التي تم بناؤها من البداية وحتى هذه اللحظة لتقديمها لعميلكم.

---

## 🛠️ أولاً: البنية الهيكلية والتقنية للمشروع
تم بناء المشروع بأحدث التقنيات البرمجية لضمان السرعة الفائقة والأمان العالي:
* **الواجهة الأمامية (Frontend):** Next.js (App Router) مع React 19 وTypeScript وتصميم Sleek Dark Theme مخصص باستخدام TailwindCSS.
* **الخلفية وقاعدة البيانات (Backend & Database):** Supabase (PostgreSQL) لإدارة البيانات والتحقق الفوري والمعاملات الآمنة.
* **الربط مع تليجرام (Telegram Integration):** استخدام Telegram Bot API للتواصل التلقائي والتحقق من اشتراكات المستخدمين وإرسال الرسائل.
* **التأمين والحماية (Security):** التحقق من هوية الجلسات ومصداقيتها باستخدام تشفير (HMAC-SHA256) للتوقيع الرقمي لبيانات دخول تليجرام لمنع أي دخول خارجي أو وهمي.

## 📱 ثانياً: شاشات تطبيق تليجرام المصغر (Telegram Mini App)

تم تصميم واجهات مستخدم متجاوبة (Responsive) وجذابة جداً، تتألف من **5 تبويبات رئيسية** بالإضافة إلى الشاشات الانتقالية والوقائية:

### 1. شاشة البداية والتسجيل الإلزامي (Onboarding Gate)
بوابة حماية إلزامية تظهر عند أول دخول للمستخدم وتمنعه تماماً من الوصول للتطبيق أو بدء التعدين حتى يقوم بإتمام **3 مهام إجبارية**:
1. **متابعة حساب X رسمي.**
2. **إعادة نشر التغريدة المثبتة.**
3. **دعوة 3 أشخاص للتطبيق** (تعرض شريط تقدم تفاعلي لحساب المدعوين `0/3` مع رابط الإحالة وزر نسخ سريع للرابط).
* بمجرد إتمام المهام، يفتح التطبيق فوراً وبشكل ديناميكي دون الحاجة لإعادة التحديث.

### 2. شاشة المهام (Home Screen)
* تعرض قائمة المهام النشطة (سواء تليجرام أو X أو إحالات).
* تمييز المهام المكتملة بعلامة صح خضراء وشعار "Verified".
* إتاحة رفع الروابط أو إثباتات الصور للمهام التي تتطلب مراجعة يدوية (مثل منصة X).
* إظهار شريط نسبة تقدم المستخدم الإجمالية في تفعيل حسابه.

### 3. شاشة التعدين (Mining Screen)
* مركز تعدين عملة OBSD مع عرض رسومي تفاعلي متطور لمحرك التعدين وسرعته.
* عرض سرعة التعدين الفعلية (`OBSD/hr`).
* عرض الأرصدة الثلاثة مقسمة بدقة:
  * **الرصيد الإجمالي (Total Balance):** النقاط الكلية المحققة.
  * **الرصيد المعلق (Pending Balance):** النقاط المكتسبة قبل إتمام شرط الشراء.
  * **الرصيد القابل للسحب (Ready Balance):** النقاط التي يمكن للمستخدم سحبها فعلياً.
* حاسبة فورية تعرض قيمة الرصيد بالدولار الأمريكي بناءً على سعر العملة المحدد من الإدارة.
* تفعيل مضاعف سرعة التعدين بنسبة **200% (Double Velocity)** تلقائياً عند تفعيل حساب المستخدم.

### 4. شاشة المحفظة والسحب (Wallet Screen)
* **ربط المحفظة:** إدخال عنوان محفظة Polygon Web3 الخاصة بالمستخدم.
* **شرط الشراء الفني (Connect Node):** رفع إثبات عملية شراء النقاط بقيمة محددة (مثل $3) لإلغاء قفل السحب وتفعيل الرصيد المعلق.
* **فترة تعليق السحب (Cooldown Timer):** عند موافقة الأدمن على الشراء، يظهر عداد تنازلي دقيق بالثواني يحسب فترة تعليق السحب الأمنية المحددة، مع قفل زر السحب حتى انتهاء المهلة.
* **تقديم طلبات السحب:** إمكانية تقديم طلب سحب رصيد OBSD لعنوان المحفظة عند بلوغ الحد الأدنى للسحب.

### 5. شاشة الإحالات والشبكة (Nodes Screen)
* عرض إحصائيات الإحالات وعدد المستخدمين المدعوين الحقيقيين.
* توفير رابط الإحالة الشخصي الفريد للمستخدم مع زر نسخ سريع بلمسة واحدة.
* رسم بياني جمالي يوضح نشاط شبكة التعدين الأسبوعي.
* تفاصيل الملف الشخصي للمستخدم والـ Telegram ID الخاص به.

### 6. شاشة الصدارة والرتب (Ranks Screen)
* لوحة متكاملة تعرض قائمة المتصدرين (Leaderboard) لأعلى المستخدمين رصيداً.
* عرض بطاقات مخصصة للمراكز الثلاثة الأولى (Top 3) بالترتيب البرونزي والفضي والذهبي.
* عرض الترتيب الحالي للمستخدم نفسه ونقاطه أسفل الشاشة لمعرفة موقعه التنافسي.

### 7. شاشة الحظر الكامل (Frozen Account Block Screen)
* واجهة أمنية كاملة تظهر للمستخدمين الذين تم تجميد حساباتهم من قبل الإدارة بسبب الغش أو مخالفة الشروط، وتمنعهم من تصفح أي جزء من التطبيق.

## 💻 ثالثاً: لوحة التحكم الإدارية (Admin Dashboard)

لوحة تحكم آمنة ومحمية بكلمة مرور مشفرة لإدارة كافة عمليات النظام:

### 1. إحصائيات النظام الفورية (Live Operations Status)
* عرض إجمالي المستخدمين المسجلين حقيقياً.
* عدد المهام النشطة المعروضة للمستخدمين.
* عدد إثباتات المهام المعلقة بانتظار المراجعة.
* عدد طلبات تفعيل شرط الشراء المعلقة بانتظار المراجعة.

### 2. إدارة المهام (Task CRUD)
* إضافة مهام جديدة وتحديد المنصة (Telegram / X / Referral).
* تحديد قيمة المكافأة للمهمة بالنقاط، ورابط التوجه للمهمة.
* تفعيل/تعطيل شرط رفع إثبات إكمال المهمة (صورة أو رابط).
* تعديل وحفظ بيانات المهام الحالية، أو أرشفتها.

### 3. إدارة المستخدمين وتجميد الحسابات (Users Management)
* قائمة بالمسجلين وتواريخ انضمامهم وأرصدتهم.
* زر تجميد/إلغاء تجميد الحساب (`Freeze`/`Unfreeze`) فوري للتحكم الفوري بأي مستخدم مسيء.

### 4. مراجعة الإثباتات والطلبات (Approvals & Reviews)
* **إثباتات المهام:** عرض إثباتات مهام X (الروابط والصور) والموافقة عليها لمنح النقاط للمستخدم أو رفضها.
* **إثباتات الشراء:** مراجعة إثباتات الدفع لتفعيل شرط الشراء والموافقة عليها (لتحرير الرصيد المعلق وبدء فترة الانتظار) أو رفضها.
* **طلبات السحب (Withdrawals):** مراجعة طلبات السحب المقدمة من الأعضاء، وعرض عناوين محافظهم وقيم السحب، والموافقة عليها (لتعليمها كمدفوعة Paid) أو رفضها.

### 5. الإعدادات العامة للنظام (System Settings)
لوحة تحكم مرنة لتعديل ثوابت النظام برمشة عين:
* تعديل الحد الأدنى للنقاط المطلوبة لتقديم طلب السحب.
* تعديل القيمة المطلوبة لتفعيل شرط الشراء بالدولار الأمريكي.
* تعديل سعر صرف عملة OBSD مقابل الدولار لتحديث الحسابات تلقائياً.
* تفعيل أو تعطيل شرط الشراء الإلزامي بضغطة زر واحدة لجميع المستخدمين.
* تعديل عدد أيام فترة تعليق السحب الأمنية (Withdrawal lock days) بعد الموافقة على الشراء.

## 🛡️ رابعاً: آليات الحماية والأمان (Security & Anti-Cheat)

1. **التحقق من جلسة تليجرام:** فحص التوقيع الرقمي الفوري (HMAC) لبيانات الدخول الصادرة من تطبيق تليجرام للتأكد من أن الطلبات قادمة من تليجرام حقيقي وليس بوت خارجي.
2. **منع تكرار مكافآت المهام:** وضع قيد فريد في قاعدة البيانات (Unique Constraint) يمنع المستخدم من المطالبة بنقاط نفس المهمة مرتين نهائياً.
3. **مكافحة الغش التلقائية (Anti-Cheat Cron Job):** فحص يومي آمن يتواصل مع Telegram API للتأكد من بقاء المستخدم داخل القنوات والمجموعات المشترك بها، وفي حال مغادرته يتم تجميد حسابه فوراً وتصفير رصيده المتاح للتعدين.
4. **حظر السحب للغشاشين والمجمدين:** فحص أمني صارم في الخادم (Backend Layer) يرفض معالجة أي معاملة سحب أو مهمة قادمة من حساب تم تجميده.
